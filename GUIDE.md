# 使用說明 · 物料管理平台

這是一份**功能與操作導覽**,以使用者視角說明這個平台能做什麼、每個畫面怎麼操作。技術細節請見 [`ANSWERS.md`](./ANSWERS.md);專案架構講解請見 [`README.md`](./README.md) 與 [`ARCHITECTURE.md`](./ARCHITECTURE.md)。

> 線上版本可直接操作:**https://material-editor-d1.zeabur.app**

這個平台用兩個視角呈現同一份料件資料:**材質編輯器**站在「估價 / 設計」的角度,讓你調整每個零件的材質與價格、即時看到 BOM 總估價的變化;**庫存看板 WMS** 站在「倉儲 / 監控」的角度,把同一批料件的庫存水位、價值分布與即時異動畫成儀表板。兩個視角共用同一份後端資料,所以在哪一邊看到的零件都是同一筆,不會各說各話。

整個操作的核心體驗是「**即時**」:在編輯器改一個值,清單與 BOM 同一瞬間跟著變;看板上的庫存數字則由後端持續推播、自動跳動。你不需要手動按「儲存」或「刷新」就能看到最新狀態。

## 整體介面

不論在哪個工作區,畫面都維持同一套外框,分成三塊:

- **左側欄(深色,固定)** — 永遠在最左邊,負責切換兩個工作區、顯示後端連線狀態、切換深 / 淺色主題。它不會隨工作區改變,是你的固定導覽。
- **頂部列** — 顯示目前所在頁面的名稱與說明、右側有即時的 SKU 數量與庫存總值、以及一個「重新載入」按鈕。
- **主內容區** — 佔畫面最大區域,依左側選的工作區,顯示「材質編輯器」或「庫存看板」的內容。

### 左側欄

側欄上半是兩個工作區的切換按鈕,目前所在的工作區會以橘色高亮;下半則是系統狀態與偏好設定。

| 元素 | 說明 |
|---|---|
| **材質編輯器** | 切換到估價 / 材質編輯視角(預設進入的頁面)。 |
| **庫存看板 WMS** | 切換到倉儲視角;若有料件低於安全庫存,按鈕右側會出現紅色數字,一眼提示目前有幾項需補貨,不必進去看也知道有狀況。 |
| **後端已連線** | 位於左下角,綠點閃爍代表與後端連線正常,旁邊顯示最後一次同步的時間,讓你確認資料是新的。 |
| **深色 / 淺色模式** | 最左下角,點一下即可在深色與淺色主題間切換;側欄本身恆為深色以維持導覽辨識度。 |

### 頂部列

頂部列讓你隨時掌握「現在在哪、整體規模多大」,不必捲到看板也能看到關鍵數字。

- **頁面標題**：顯示目前工作區的名稱與一句說明。
- **SKU / 金額**：右側的小資訊塊,顯示目前料件種類數(SKU)以及即時的庫存總價值——這兩個數字會隨編輯與庫存推播持續更新。
- **重新載入**：手動重新向後端抓取最新料件資料;一般情況不需要按(資料會自動同步),適合在你懷疑資料過舊時強制刷新。

## 工作區一:材質編輯器

這是用來編輯零件材質與價格、並即時看到專案估價變化的工作區。畫面分成三區——左上是零件清單、右上是編輯器、下方是 BOM 總表——它們**共用同一份資料**,所以你在任何一區改動,另外兩區會在同一瞬間跟著更新。實務上你會這樣用:在左邊點一個零件 → 在右邊調它的顏色 / 單價 / 加成 → 下方 BOM 的小計與總價立刻反映。

### 1. 模型零件清單(左)

這是所有零件的列表,也是你選取「要編輯哪一個」的入口。

- 每一列顯示該零件的**名稱、分類、單項小計、顏色方塊**,讓你不必展開就看到重點。
- 點任一列即選取該零件,右側編輯器會載入它的完整資料;**被選中的列會以橘色高亮**,方便你知道目前在編誰。
- 當你在編輯器改動價格或加成,這裡對應列的小計會即時更新——這就是「三區同步」最直接的體現。

### 2. 材質與屬性編輯器(右)

選取一個零件後,這裡載入它的詳細屬性供你編輯。**每一次變更都會即時反映到左側清單與下方 BOM**,不需要按儲存:

| 欄位 | 操作 | 範圍 |
|---|---|---|
| **材質顏色(Albedo)** | 點色塊開調色盤,或直接輸入十六進位色碼 | `#RRGGBB` |
| **基礎單價(Base Price)** | 數字輸入框,可用上下鍵微調 | ≥ 0,每步 10 |
| **材質複雜度加成(Multiplier)** | 拖動滑桿 | 1.0× ~ 5.0×,每步 0.1 |

編輯器頂部與底部還有兩個你會一直注意到的提示:

- **「已同步」標示(右上角)** —— 告訴你目前變更的儲存狀態:平時顯示「已同步」代表後端已收到;儲存中會顯示轉圈;萬一儲存失敗,會出現「重試」按鈕讓你重送。你不用主動存檔,系統會在背景幫你存。
- **「即時小計(Line Total)」(底部)** —— 等於「基礎單價 × 加成」,是這個零件的單項估價。拖動上面的滑桿時,這個數字會即時跳動,讓你邊調邊看價格。

> 💡 拖動滑桿時畫面**立即**更新,但系統不會每次都打後端——它會等你停下約 0.3 秒後才把變更存回。所以你可以放心快速拖曳,既不會卡頓、也不會對後端送出一大堆請求,停手後才安靜地存一次。

### 3. BOM 物料清單預估(下)

BOM(Bill of Materials,物料清單)把所有零件的估價攤平成一張表,讓你綜觀整個專案的成本結構。

- 表格逐列顯示每個零件的**名稱、色碼、單價 × 加成、小計**。
- 點任一列也可以選取該零件(與左側清單連動,選哪個都會高亮),方便你從成本表直接跳去編輯某個零件。
- 表格最下方的「**專案總估價(Grand Total)**」是所有零件小計的加總;只要你在編輯器改了任何一個零件,這個總價就會即時重算——這是驗證「即時同步」最有感的地方。

## 工作區二:庫存看板 WMS

這是倉儲管理(WMS)視角的儀表板,呈現的是**和編輯器同一批料件**,但關注點換成「庫存夠不夠、價值多少、哪裡有異動」。它的特色是**會自己動**:後端模擬感測器持續回報庫存變化,看板的數字、圖表與明細不需要你刷新就會即時跳動,模擬真實倉儲現場的即時監控。畫面由上而下分成 KPI、圖表 / 即時流、庫存明細三層。

### 上方:KPI 卡片

四張卡片是整個倉儲狀態的「一眼摘要」,放在最上方讓你進來就掌握全局:

| 卡片 | 意義 |
|---|---|
| **總料件數** | 料件種類(SKU)數量,代表管理的品項廣度。 |
| **庫存總量** | 所有料件的庫存件數總和,反映整體存量。 |
| **庫存總價值** | Σ(單價 × 加成 × 庫存),代表目前壓在庫存上的金額。 |
| **低於安全庫存** | 庫存已低於安全水位、需要補貨的料件數;一旦有缺貨,這張卡片會轉為紅色警示,提醒你該採取行動。 |

### 中間左:各分類庫存價值佔比

把庫存價值依分類(結構組件、外部面板、電子元件…)拆開,用**橫條圖**呈現各分類的庫存價值,長度越長代表該分類佔用的庫存金額越多。這讓你快速看出「錢主要壓在哪一類料件上」,是盤點與採購決策的參考。

### 中間右:IoT 即時數據流

這一區是看板「即時」特性的來源與控制台,模擬倉儲現場感測器每 1.5 秒回報一次庫存異動,看板隨之刷新。

- **狀態燈**:`LIVE`(綠燈閃爍,資料流進行中)/ `RECONNECT…`(連線中斷、正在自動重連)/ `PAUSED`(已手動暫停)。看燈就知道資料流目前的狀態。
- **暫停資料流 / 啟動資料流** 按鈕:手動開關即時推播。想仔細看某個瞬間的數字時,按「暫停」可以凍結畫面;看完再「啟動」恢復。
- **事件流(Event Log)**:列出最近幾筆庫存異動,**綠色 `+` 為入庫、紅色 `−` 為出庫**,每筆都附時間戳,像一條即時的庫存流水帳。

> 💡 這裡的庫存變化是**後端**產生、寫入資料庫後才推播給畫面的,不是前端自己假裝的。所以即使你重新整理頁面,庫存數字也會保留(不會跳回原值);而且若你同時開兩個分頁,兩邊會收到同一份推播、保持一致。

### 下方:庫存明細(Inventory)

最下方的表格把每一筆料件的庫存狀態攤開,是 KPI 摘要的細節展開——當上方「低於安全庫存」亮紅燈時,你就到這裡找出是哪幾筆、缺多少:

| 欄位 | 說明 |
|---|---|
| **料件 / 儲位** | 名稱、分類與儲位代碼(如 `A-01-03`)。 |
| **庫存量** | 目前庫存;低於安全庫存時顯示紅色。 |
| **水位** | 進度條呈現庫存相對安全庫存的水位。 |
| **安全庫存** | 補貨警戒線。 |
| **庫存價值** | 該料件的庫存總值。 |
| **狀態** | 綠色「正常」或紅色「需補貨」標籤;需補貨的整列以紅底標示。 |

## 常見操作情境

如果你只想快速體驗這個平台的重點,照下面這幾個情境操作最有感:

- **想試三區塊即時同步** → 進「材質編輯器」,拖動某零件的 Multiplier 滑桿,同時盯著左側清單該列的價格與下方 BOM 的總價——它們會跟著滑桿一起跳動,這就是「改一處、三處同步」。
- **想看即時數據** → 進「庫存看板 WMS」,觀察事件流持續滾動、KPI 數字與長條圖隨之刷新;想看清楚某個瞬間時,按「暫停資料流」凍結畫面,看完再啟動。
- **想驗證資料真的存進後端** → 在編輯器改某零件的單價,然後重新整理整個頁面;改動仍然在(因為已存回後端),代表不是只改在你這台瀏覽器。
- **想換配色** → 左下角點「深色 / 淺色模式」切換主題,適合不同光線環境或個人偏好。
